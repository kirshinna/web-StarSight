Техническое задание на проект: StarSight

1. Цель проекта

Создание веб-сервиса для оценки условий астрономических наблюдений. Система помогает любителям астрономии определить оптимальное время для наблюдения за звёздами и Луной, используя прогноз облачности и фазу Луны, визуализируя данные в виде тепловой карты качества ночи.

2. Роли пользователей

Гость: просмотр формы для ввода города и дат, просмотр тепловой карты и сводной информации о ночных условиях, просмотр и заполнение дневника наблюдений.

Зарегистрированный пользователь: возможность вести личные заметки наблюдений.

3. Модели данных (сущности)

Observation - Запись наблюдений.
Поля: date, title, description, image, created_at.

DayForecast — Данные по каждому дню прогноза.
Поля: city, date, cloudcover_avg, moonphase_avg.

ObservationDay — День наблюдений, представляет один день, за который пользователь ведёт запись.
Поля:date, city, created_at

4. Ключевой функционал (User Stories)

Несколько страниц: Пользователь может переключаться между страницами.

Запрос прогноза: Пользователь вводит город и диапазон дат, система формирует прогноз облачности, фаз Луны по ночам, видимости.

Построение тепловой карты: Для каждого дня и ночных часов система вычисляет «вес» (score) условий наблюдения и визуализирует данные в виде цветной карты.

Подробные данные по часам: Пользователь может раскрыть блок, где указаны данные по каждому часу ночи.

Дневник наблюдений: Пользователь может добавлять записи и просматривать их.

5. Внешние интеграции / Аналитика

API: Используется Visual Crossing Weather API для получения прогноза погоды и астрономических данных (фазы Луны, почасовые облачность и освещённость).

Аналитика и визуализация: Используются библиотеки Pandas для обработки данных и Matplotlib / Seaborn для построения тепловых карт и графиков.
